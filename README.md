Проектирование типов

Решено было начать с типов для браузерной валидации и форм, далее для создания отдельных "схем".

по фалам добавленным (понимание от лица (Максимова)):
validity.ts - браузерная валидация, инпуты и глобально какие существуют html элементы которые можно валидировать
form.ts - для работы с формой "отдельно от inputs"
field.ts - создатель полей схемы
core/Form.ts - Fluent API для объявления правил валидации, поддерживащее цепочка методов
types/index.ts - ре-экспорт типов для создания единой точки входа
index.ts - публичная точка входа.

Второй этап

переработан код, сделанна реализация на 3 классах

первый это FormConnector(), он связынввает основной класс с формой в html и получает все поля в виде массива.
второй это Validator(), он отвечает, валидирует одно поле с помощью нескольких экземпляров класса.
и основной это V, его импортирует ипользователь, в него вшиты интерфесы для FormConnector() и Validator(), фактически это интерфейс для поользователя.
